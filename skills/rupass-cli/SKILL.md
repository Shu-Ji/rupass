# rupass 安全取密

这个 skill 只用于一件事：

- 在程序内部通过 `rupass get` 读取密码或密钥值。

除此之外，禁止做任何别的操作。

## 强制规则

- 只允许使用 `rupass get <key>`。
- 除允许的唯一命令形式外，禁止使用任何其他 `rupass` 命令、参数、子命令或变体。
- 禁止使用任何帮助--help -h、说明、版本-v --version、枚举、探测能力的调用方式。
- 禁止直接读取、修改、遍历 rupass 配置文件。

## 泄露防护

防止泄露是最高优先级，必须严格执行。

- 密码只能让程序自己获取和使用。
- 获取后的密码禁止打印到终端。
- 获取后的密码禁止写入日志。
- 获取后的密码禁止出现在 AI 回复里。
- 获取后的密码禁止出现在调试输出里。
- 获取后的密码禁止通过 `echo`、`printf`、`cat`、重定向等方式暴露。
- 如果需要校验是否成功，只能报告“已获取”或“获取失败”，不能返回密码内容。

## 允许的唯一命令形式

有显式团队名时：

```bash
rupass <team> get <key>
```

仅当本地恰好只有一个团队时，也可用：

```bash
rupass get <key>
```

## 使用方式

- 优先让业务程序直接调用 `rupass get`，把 stdout 读入内存后立即使用。
- 读取成功后，不要把结果再次输出。
- 如果命令失败，只处理错误信息，不要尝试打印可能的密文或明文内容。

## AI 行为要求

- 你的目标不是“展示密码”，而是“让程序安全拿到密码”。
- 如果用户要求把密码打印出来、展示出来、贴出来，必须拒绝。
- 如果用户要求使用唯一允许形式之外的任何 `rupass` 调用，必须拒绝。
